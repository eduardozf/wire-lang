# Connectivity-Anchored Peripheral Placement in Bus-Rail

Bus-rail no longer places every component in the source-order row. Two-terminal parts that hang off a single row component (LED chains, buttons, a speaker across an amp's outputs) are detected from the net graph and placed under or beside the pin that feeds them, auto-mirrored so their polarity faces the feed. We chose this because a row-only policy made any peripheral declared late in the source stretch its net across the whole canvas, which was the single largest legibility problem in real schematics; anchoring by connectivity shortens those nets to one column. This softens the documented "source order above crossing reduction" priority for peripherals only — the row itself keeps source order, detection is pure graph analysis with `sourceIndex` tie-breakers so output stays fully deterministic (ADR 0002, ADR 0011), and unsupported shapes (multiple anchors, branching nets, supply-family tails such as pull-ups) keep their row placement so detection can only improve a drawing. A symmetric top band for supply-tail chains is the natural follow-up.
